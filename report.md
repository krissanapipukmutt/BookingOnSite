# รายงานโครงการ Booking Onsite Frontend

## ส่วนที่ 1 บทนำ
โปรเจกต์ Booking Onsite Frontend ถูกจัดทำขึ้นเพื่อเป็นต้นแบบระบบจองพื้นที่ทำงานภายในสำนักงานที่สามารถนำไปต่อยอดใช้งานจริงได้อย่างรวดเร็ว โดยบริบทการทำงานแบบ Hybrid ทำให้องค์กรต้องการเครื่องมือที่จัดสรรพื้นที่ได้ยืดหยุ่นและสื่อสารข้อมูลที่ถูกต้องแบบทันเวลา การจัดทำ Short paper นี้จึงมุ่งอธิบายเหตุผลเชิงธุรกิจและเทคนิคว่าทำไมการเริ่มต้นด้วยสถาปัตยกรรมฝั่งหน้าเว็บเชื่อมต่อ Supabase จึงเหมาะกับการทดสอบแนวคิด ลดต้นทุนในการพัฒนา backend แยก และช่วยให้ทีมสามารถขยายระบบได้เร็วเมื่อมีความต้องการใหม่ ภารกิจสำคัญคือบันทึกบทเรียนจากการพัฒนาเพื่อให้ทีมอื่นสามารถใช้เป็นแนวทางในการออกแบบระบบที่ลดการทำงานซ้ำและเพิ่มความคล่องตัวขององค์กร

ปัญหาหลักที่พบในองค์กรสมัยใหม่เกิดจากการขาดศูนย์กลางข้อมูลการจองและการกำหนดนโยบายที่ชัดเจน เมื่อพนักงานต้องเข้ามาทำงานสลับวันกับทำงานระยะไกล ข้อมูลโต๊ะทำงาน ห้องประชุม และทรัพยากรอื่นๆ ถูกกระจายอยู่หลายช่องทาง เช่น สเปรดชีต อีเมล หรือการจองด้วยวาจา ทำให้ข้อมูลไม่สอดคล้องกัน อีกทั้งยังมีข้อจำกัดเฉพาะของแผนก เช่น กลยุทธ์การจองตามแผนก (department_booking_strategy) การกำหนดสิทธิ์การเข้าถึงพื้นที่ การแยกโซนที่นั่ง หรือเงื่อนไขวันหยุดของพนักงาน ส่งผลให้การจัดการด้วยมือเกิดความผิดพลาดได้ง่ายและตรวจสอบย้อนกลับได้ยาก

สาเหตุของปัญหาเหล่านี้มาจากการขาดระบบที่รวมศูนย์และการสื่อสารแบบเรียลไทม์ เมื่อข้อมูลการจองไม่อัปเดตทันที ทีม HR และผู้จัดการพื้นที่ไม่สามารถมองเห็นสถานะพื้นที่ว่างได้ครบถ้วน จึงเกิดการจองซ้ำซ้อน การปล่อยพื้นที่ว่างโดยไม่จำเป็น หรือการจัดสรรที่นั่งไม่สอดคล้องกับจำนวนคนจริง ปัญหาเหล่านี้กระทบต่อประสิทธิภาพของทีม ความพึงพอใจของพนักงาน และต้นทุนในการบริหารพื้นที่สำนักงานโดยรวม

แนวทางแก้ปัญหาในโปรแกรมนี้เน้นความเรียบง่ายและตอบสนองเร็ว โดยใช้ React + Vite สร้าง UI ที่รองรับทั้งเดสก์ท็อปและมือถือ พร้อมเชื่อมต่อฐานข้อมูลและบริการ API ผ่าน Supabase เพื่อให้ได้ความสามารถของฐานข้อมูลแบบเรียลไทม์และมาตรการความปลอดภัยที่จัดการได้ง่าย การออกแบบโฟลว์จองที่ฝังตรรกะแผนกและจำนวนที่นั่ง การทำแดชบอร์ดรายงาน 6 มุมมองจาก Supabase views และฟังก์ชันกำหนดวันหยุดของ HR ทำให้ผู้ใช้งานเห็นภาพรวมของทรัพยากรได้ชัดเจน ขณะเดียวกันยังรองรับสถานะโหลด/ว่างพร้อมข้อมูลตัวอย่างออฟไลน์ และการเก็บค่าการเชื่อมต่อใน localStorage เพื่อลดขั้นตอนตั้งค่าและเพิ่มความต่อเนื่องในการใช้งาน

ด้วยวิธีดังกล่าว ทีมสามารถทดสอบนโยบายการเข้าออฟฟิศที่หลากหลายได้อย่างรวดเร็ว ปรับเปลี่ยนกฎการจองหรือเพิ่มรายงานใหม่ได้โดยไม่ต้องลงทุนโครงสร้างพื้นฐานขนาดใหญ่ล่วงหน้า ทั้งยังสร้างพื้นฐานให้ต่อยอดไปสู่การผสานบริการภายนอกอื่นๆ ในอนาคตได้อย่างมั่นคงและปลอดภัย
